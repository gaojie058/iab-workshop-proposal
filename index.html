Overview

Motivation

LLM-powered agents now autonomously plan, execute, and iterate on complex tasks across software engineering, scientific research, and daily workflows. Yet current evaluation relies almost exclusively on automated benchmarks that measure whether an agent succeeded, not how it behaved, why it failed, or how humans interpreted its actions along the way.

Agentic systems continuously generate execution traces, reasoning chains, tool-use logs, and error signals. Understanding agent behavior from this data can help us diagnose failures, predict risk, optimize agent design, and calibrate human trust. Yet the analytical methods remain fragmented: ML measures outcomes; HCI and the social sciences offer complementary tools for analyzing processes.

The Interpreting Agent Behavior (IAB) workshop proposes to bridge this methodological gap, establishing a common vocabulary for describing agent behaviors, catalyzing open behavioral datasets, and seeding a recurring community that pushes agent interpretation from ad hoc debugging toward a rigorous, reproducible science.

Scope

Proposed Research Questions

We plan to investigate agent behavior from three complementary perspectives.

Agent Behavior

  • Decision-making under uncertainty
  • Failure cascades and error recovery
  • Multi-agent coordination and emergent behaviors
  • Behavioral differences across LLM backbones

Human Response

  • Trust calibration and over-reliance
  • Verification strategies for agent errors
  • Mental model formation and updating
  • Cognitive sense-making processes

Human–Agent Interaction

  • Systematic coding of interaction traces
  • Co-evolution of instructions and responses
  • Interface effects on behavior and trust
  • Communication breakdowns and task failures

Submissions

Planned Call for Papers

If accepted, we will welcome submissions on new results, negative findings, benchmarks, position papers, and system demos. All submissions would be non-archival.

Long Papers

Up to 9 pages + references. Empirical studies, benchmarks, or comprehensive analyses.

Short Papers

Up to 4 pages + references. Position papers, demos, preliminary findings, or negative results.

Review Process

COLM 2026 style. Single-anonymous. At least 2 reviews per submission via OpenReview.
